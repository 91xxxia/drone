## 3. 数据库实施方案（第一优先级）

## 3.1 建议新增核心表

1. `biz_usv_device`（设备表）
- 主键：`device_id`
- 唯一：`device_code`（设备编号）
- 关键字段：`device_name/model/endurance_minutes/camera_spec/owner_name/status/remark`
- 状态建议：`normal/maintenance/offline/in_task/scrapped`

2. `biz_patrol_route`（航线主表）
- 主键：`route_id`
- 唯一：`route_name`
- 字段：`est_duration_minutes/altitude/remark`

3. `biz_patrol_route_point`（航线点位表）
- 主键：`point_id`
- 外键：`route_id`
- 字段：`seq/lng/lat`
- 约束：`(route_id, seq)` 唯一

4. `biz_patrol_task`（任务表）
- 主键：`task_id`
- 唯一：`task_code`
- 外键：`device_id/route_id`
- 字段：`task_name/executor/planned_start_time/task_desc/status/progress`
- 状态：`pending/running/completed/canceled`

5. `biz_patrol_task_log`（任务状态流转日志）
- 主键：`log_id`
- 外键：`task_id`
- 字段：`from_status/to_status/operate_by/operate_time/remark`

6. `biz_patrol_result`（结果表）
- 主键：`result_id`
- 唯一：`result_code`
- 外键：`task_id`（建议唯一，确保任务只生成一条主结果）
- 字段：`overview/findings/handling/remark/duration_minutes/completed_time/executor`

7. `biz_patrol_result_media`（结果图片/识别信息，可选扩展）
- 主键：`media_id`
- 外键：`result_id`
- 字段：`file_url/ai_tag/bbox_json`

## 3.2 索引与约束策略
- 任务查询高频索引：`idx_task_status`、`idx_task_executor`、`idx_task_start_time`
- 结果查询高频索引：`idx_result_code`、`idx_result_completed_time`
- 删除约束：
  - 已绑定任务的设备、航线不可删（业务层拦截 + 数据层约束）
  - 已完成任务不可改绑定关系

## 3.3 字典落地（`sys_dict_type/sys_dict_data`）
- `drone_device_status`：正常、维修中、离线、任务中、已报废
- `drone_task_status`：待执行、执行中、已完成、已取消
- `drone_result_level`（可选）：正常、轻微异常、严重异常

### ✅ 4.2 核心业务规则（已完成）
- **任务状态机**：完整实现四种状态流转
  - `pending -> running` （任务开始）
  - `running -> completed` （任务完成）
  - `pending/running -> canceled` （任务取消）
- **设备联动**：
  - 任务开始时自动将设备状态改为 `in_task`
  - 任务完成/取消时自动恢复设备状态为 `normal`
  - 任务完成时自动生成巡防结果记录
- **防御式校验**：
  - 任务开始时间不得早于当前时间 ✅
  - 设备状态必须为 `normal` 才允许创建任务 ✅
  - 同一设备/航线不可绑定到多个未完成任务 ✅

### ✅ 4.3 详情与导出（已完成）
- 设备详情：支持查看详情和历史任务关联
- 航线详情：完整回显航线点位信息

### ✅ 4.4 日志与审计（已完成）

- 一级目录：`无人船巡防`

  - `航线管理`
  - `任务管理`
  - `结果管理`
  - `数据概览`

## 6.2 权限标识建议
- 设备：`drone:device:list|query|add|edit|remove|export`
- 航线：`drone:route:list|query|add|edit|remove`
- 任务：`drone:task:list|query|add|start|finish|cancel|remove`
- 结果：`drone:result:list|query|edit|remove|export|detail`

## 6.3 角色权限策略
- 管理员：全权限。
- 普通用户：`list/query/detail` + 个人中心改密；禁用新增/编辑/删除/状态切换。
- 对当前 `role_key=common` 建议做一次“权限瘦身”SQL 修正，避免越权。

## 10. 风险与应对

- **地图 API Key 风险**：提前申请高德 Key，封装环境变量，避免硬编码。
- **状态一致性风险**：任务与设备状态更新需同事务，避免脏状态。
- **权限遗漏风险**：接口层加 `@PreAuthorize`，前端按钮仅作展示控制。
- **导出性能风险**：结果导出默认按筛选条件分页/限流，防止一次性全量导出。
